# Industrial Upgrade Goal Plan

Use this reference when improving or executing the skill as an industrial ecommerce image production system.

## Goal

Build SellerPilot Product Image Industrial into a goal-driven commercial visual production system: product truth, market context, audience positioning, competitive/bestseller learning, creative strategy, photography treatment, layout design, personalized GPT built-in image generation prompt delivery, QA, review, and performance learning.

The final production handoff may be a detailed personalized prompt, plus a request pack only when fallback or audit evidence is needed. The prompt is not the starting point. It is the final production work order synthesized after analysis, sketches, self-review, and role collaboration.

## Industrial Principles

- Start from a run goal, not from a prompt.
- Separate strategy, creative direction, photography treatment, copy, layout, generation, and QA.
- Convert each role's judgment into artifacts that can be reviewed and reused.
- Use sketches and low-fidelity drafts to reduce final-generation waste.
- Generate final prompts only after product truth, platform fit, audience, scene logic, and layout intent are locked.
- Treat the final generation prompt as a personalized execution brief for a specific product, platform, image role, audience, season, and market moment.
- Fail early when the workflow lacks source evidence, scene assets, platform fit, or buyer-facing rationale.
- Never claim guaranteed CTR, CVR, ROAS, ranking, or sales lift; optimize for click/conversion intent and review with actual data when available.

## Role Model

Use these roles as a collaboration model. A single Codex run may perform several roles, but the outputs must stay separate:

- Product Truth Analyst: source evidence, facts, identity lock, unsupported claims.
- Source Retoucher: source quality, enhancement, detail clarity, multi-source consistency.
- Marketplace Strategist: platform, country, category, season, event, and platform-profile overlay.
- Audience Planner: buyer motivations, objections, occasions, aesthetic preferences, language.
- Competitive Pattern Analyst: bestseller patterns, competitor strengths/weaknesses, safe borrowing.
- Commerce Planner: image-set architecture, buyer question per image, conversion-intent task.
- Creative Director: creative concept, tone, visual memory point, differentiation.
- Commercial Photography Director: lens, angle, lighting, color temperature, model/prop/scene treatment.
- Copy Director: buyer-facing wording, platform language density, claim boundaries.
- Layout Designer: wireframe, grid, typography, hierarchy, VI consistency, mobile readability.
- Generation Producer: final generation prompts, retry strategy, asset status, and blocked generation notes when execution is unavailable.
- QA Director: gates, self-review, identity consistency, scene authenticity, export package.
- Growth Analyst: post-run learning, performance notes, next-test hypotheses.

## Upgrade Phases

### Phase 1: Strategy and Prompt Handoff Backbone

Add durable artifacts for:

- `goal-contract.yaml`
- `commerce-strategy-brief.yaml`
- `creative-direction-brief.yaml`
- `photography-treatment.yaml`
- `layout-wireframes.yaml`
- `final-prompt-delivery.md`
- prompt/readiness QA

Done when a new run skeleton creates these artifacts and the master workflow requires them before final product-bearing generation.

### Phase 2: Sketch-To-Final Loop

Require low-fidelity image-role sketches before final prompt delivery:

- thumbnail sketch per image role
- scene sketch for scene images
- layout sketch for text-heavy images
- self-review notes
- revision reason

Done when the skill can block final prompt delivery if sketches are absent for complex image sets.

### Phase 3: Photography Treatment Depth

Make photography treatment explicit:

- lens / crop / camera height
- lighting direction and color temperature
- background/scene material
- model pose or body relationship
- product placement and scale
- detail capture plan

Done when scene/product prompts include photography fields, not just generic scene words.

### Phase 4: Layout Design System

Add layout rules:

- grid and safe zones
- typography hierarchy
- product coverage percentage
- mobile thumbnail readability
- platform-specific density
- VI and color system

Done when layout QA can fail weak hierarchy or unreadable mobile image text.

### Phase 5: Review And Learning Loop

Add post-production learning:

- rejected pattern log
- failed prompt diagnosis
- performance hypothesis
- A/B next-test note
- platform/category reusable learning candidates

Done when each export package includes what should be learned, not only what was produced.

## Definition Of Done

An industrial run is complete only when:

- the run goal and commercial objective are explicit.
- source/product facts are separated from assumptions.
- market/platform/category/season context is captured.
- audience and buyer objections are mapped.
- each image has a buyer question and conversion-intent task.
- visual concept, photography treatment, and layout wireframe exist before final prompt delivery.
- final generation prompt is personalized per image role and includes identity, scene, camera, light, layout, copy, QA, and retry policy.
- generated/photo assets pass identity, scene authenticity, marketing, layout, and export gates.
- final output includes independent files, review surface, revision history, and a learning note.
